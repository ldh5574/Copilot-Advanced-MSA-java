# 1. 문제 정의 (Problem Definition)

## 학습 목표
- 요구사항을 정확히 이해하고 분석하는 방법 습득
- Copilot을 활용하여 요구사항을 코드로 변환하는 첫 단계 학습

---

## 교육 내용

### 1.1 요구사항 확인의 중요성
개발의 첫 단계는 **무엇을 만들어야 하는지** 정확히 파악하는 것입니다.

#### 잘못된 예시
```
"사용자 관리 기능 만들어주세요"
```

#### 올바른 예시
```
"사용자 관리 기능을 만들어야 합니다.
- 사용자 등록: 이름, 이메일, 비밀번호 입력
- 사용자 조회: ID로 단일 조회, 전체 목록 조회
- 사용자 삭제: ID로 삭제
- 이메일 중복 체크 필요
- 비밀번호는 6자 이상"
```

### 1.2 MSA 프로젝트 요구사항

본 교육 프로젝트의 요구사항입니다:

#### 시스템 개요
간단한 E-Commerce 시스템을 MSA로 구현

#### 서비스 구성
| 서비스 | 포트 | 역할 |
|--------|------|------|
| API Gateway | 8080 | 라우팅, 진입점 |
| User Service | 8081 | 사용자 관리 |
| Product Service | 8082 | 상품/재고 관리 |
| Order Service | 8083 | 주문 관리 |

#### 기능 요구사항

**User Service**
- 사용자 CRUD (생성, 조회, 삭제)
- 사용자 생성 시 Kafka 이벤트 발행

**Product Service**
- 상품 CRUD
- 재고 증가/감소
- 주문 이벤트 수신 시 재고 차감

**Order Service**
- 주문 생성/조회/취소
- 주문 생성 시 Kafka 이벤트 발행

#### 비기능 요구사항
- Docker Compose로 전체 시스템 실행 가능
- H2 In-Memory DB 사용
- Kafka를 통한 비동기 통신

## 실습

### 실습 1: 요구사항 분석
1. 위 요구사항을 읽고 각 서비스 간의 관계를 다이어그램으로 그려보세요
2. Kafka 이벤트 흐름을 정리해보세요

### 실습 2: Copilot 활용
아래 프롬프트로 Copilot에게 요구사항 정리를 요청해보세요:

```
다음 요구사항을 분석하고 ERD와 API 엔드포인트를 정리해주세요:
- User Service: 사용자 등록/조회/삭제
- Product Service: 상품 등록/조회, 재고 관리
- Order Service: 주문 생성/조회/취소
각 서비스는 Kafka로 통신합니다.
```
